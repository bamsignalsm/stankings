# Stankings Brand Assets

Single source of truth for public brand surfaces on [stankings.com](https://stankings.com).

**Do not redesign logos in application code.** Drop approved master artwork into `public/images/source/`, then regenerate derived assets.

---

## Ownership

| Role | Responsibility |
|------|----------------|
| Design / founders | Approve and deliver master raster artwork |
| Engineering | Wire `BRAND` constants, run optimizer, keep CI green |
| Product apps (BamSignal, BayRight, Yike, …) | Consume their own brand modules; never hardcode Stankings asset paths |

---

## Master source (editable)

| File | Purpose |
|------|---------|
| `public/images/source/logo-master.png` | **Active** master used by the optimizer |
| `public/images/source/logo-master-v1.png` | Versioned archive of the current master |
| `public/images/source/logo-master-v2.png` | *(future)* next approved master |

When new artwork arrives:

1. Save it as `logo-master-vN.png` (next version).
2. Copy it over `logo-master.png`.
3. Run `npm run optimize:brand`.
4. Commit source + generated outputs together.

**Do not** programmatically patch text inside the PNG (e.g. “STANKINGS GROUP” → legal name). Wait for revised official artwork.

## Generated production assets (immutable)

Location: `public/images/`

| Asset | Path | Role | Typical size |
|-------|------|------|--------------|
| Primary logo | `/images/logo.webp` | Nav, auth, media kit | ~1209×288 (source aspect) |
| App icon | `/images/icon.webp` | Square mark | 512×512 |
| Favicon 32 | `/images/icon-32.webp` | Browser tab | 32×32 |
| Apple touch | `/images/icon-180.webp` | iOS / PWA | 180×180 |
| PWA / large | `/images/icon-512.webp` | Manifest maskable | 512×512 |
| Open Graph JPEG | `/images/og-image.jpg` | WhatsApp / Facebook / Telegram | 1200×630 |
| Open Graph WebP | `/images/og-image.webp` | Alternate OG | 1200×630 |

These files are **generated**. Do not edit them in Figma exports or image editors.

```
DO NOT EDIT

Generated by:

npm run optimize:brand
```

See also `public/images/GENERATED.md`.

## Generation workflow

```bash
# 1. Place approved master at:
#    public/images/source/logo-master.png

# 2. Regenerate all derived brand surfaces
npm run optimize:brand

# 3. Validate before commit
npm run verify:brand
```

Optimizer: `scripts/optimize-brand-assets.mjs`  
Validator: `scripts/verify-brand-assets.mjs`

---

## Runtime API

Application code must use:

```ts
import { BRAND } from "@/lib/brand";

BRAND.logo.src      // "/images/logo.webp"
BRAND.icon.src
BRAND.ogImage.src
BRAND.favicon.src32 / src180 / src512
```

Never import or hardcode `/images/logo.png` in production UI.

### Email / transactional templates (future)

If HTML email templates are added, they **must** consume `BRAND.logo.src` (or an absolute URL built from `SITE_URL` + `BRAND.logo.src`). Never hardcode logo filenames.

## Multi-brand note

Stankings is the holding-company site. Product brands (BamSignal, BayRight, Yike, …) will live under `src/lib/branding/brands/` when needed. Today only `stankings` is registered; `src/lib/brand.ts` re-exports it so existing imports stay stable.
