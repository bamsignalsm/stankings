# 9. Role-Based Dashboards

| workspace_key | Modules |
|---------------|---------|
| `editorial` | Drafts, Pending Review, Published, Media, Categories, Calendar, SEO, Analytics, Social |
| `public_relations` | Press releases, Media requests, Campaigns, Contacts, Announcements, Events |
| `customer_support` | Assigned/Open/Resolved tickets, Escalations, KB, Customer lookup, Notes, Performance |
| `moderation` | Reports, Photo/Listing/Profile review, Appeals, Safety, Actions history |
| `operations` | Tasks, Daily ops, Reports, Documents, Approvals, Activity |
| `engineering` | Sprint board, Issues, Docs, Deployments, Incidents, Architecture |
| `risk_ops` | Settlements, Payment exceptions, Evidence, Ops dashboard |
| `education_ops` | Admissions, Student success, Learning, Community |
| `foundation_ops` | Programmes, Volunteers, Engagement, Communications |
| `executive` | Org overview (scoped), Approvals, Reports |
| `people_ops` | Hiring funnel view (read), Onboarding checklists |
| `sales` | Pipeline, Clients, Inspections |
| `verification` | Property/Vehicle verification queues |

Role titles map to one primary `workspace_key` in the job catalogue.
